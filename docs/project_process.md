# 사물인터넷과 자율주행 프로젝트 프로세스 정리

## 1. 프로젝트 목표

사물인터넷과 자율주행 수업에서 정해진 트랙을 자율주행 차량이 10바퀴 완주하도록 만드는 것이 목표였습니다. 제공된 부품은 바퀴, 배터리, 모터, 라즈베리파이 등 차량 구동에 필요한 기본 부품들이었고, 차량 프레임은 직접 3D 모델링 후 제작해야 했습니다.

## 2. 초기 하드웨어 문제

초기 제작한 차량은 차체 길이가 길고 무게중심이 맞지 않아 회전 성능이 좋지 않았습니다. 특히 후륜 차량 특성과 무게중심 문제 때문에 자율주행 이전에 MIT App Inventor 기반 조이스틱으로 직접 조종했을 때도 코너를 안정적으로 돌기 어려웠습니다.

이 단계에서 중요한 판단은 AI 모델보다 먼저 차량 자체를 조종 가능한 플랫폼으로 만드는 것이었습니다. 사람이 조종해도 불안정한 차량은 좋은 데이터를 만들기 어렵고, 같은 조향 명령에도 차량 반응이 달라져 모델 학습에도 불리하기 때문입니다.

## 3. 차량 튜닝

남은 시간이 많지 않아 차량 프레임을 다시 설계하거나 재출력하기는 어려웠습니다. 그래서 하드웨어 전체를 다시 만드는 대신, 가능한 범위에서 차량의 무게중심을 후방으로 이동시키고 모터 제어 값을 조정했습니다.

구체적으로는 회전 시 좌우 바퀴의 속도 차이를 더 크게 주고, 필요한 경우 회전 구간에서 속도 값을 조정해 차량이 더 적극적으로 방향을 바꾸도록 했습니다. 이 과정을 통해 기존 차량 구조를 유지하면서도 사람이 조종 가능한 수준으로 차량 거동을 안정화했습니다.

## 4. 데이터 수집

차량이 어느 정도 안정된 뒤 MIT App Inventor 조이스틱 제어로 직접 주행하며 이미지와 조향 데이터를 수집했습니다. 이 데이터는 이후 자율주행 모델 학습에 사용되었습니다.

데이터 수집 과정에서는 코너에서 애매한 조향을 오래 유지하기보다, 실제로 필요한 조향 값을 명확하게 주는 방식이 모델 학습에 더 유리하다는 점을 확인했습니다. 즉, 모델이 코너 상황과 조향 명령의 관계를 더 뚜렷하게 학습할 수 있도록 데이터 품질을 신경 썼습니다.

## 5. CNN-LSTM 실험

초기에는 연속 촬영된 이미지 데이터를 활용하기 위해 CNN-LSTM 기반 하이브리드 모델을 실험했습니다. CNN으로 각 프레임의 이미지 특징을 추출하고, LSTM으로 여러 프레임의 흐름을 반영해 조향을 예측하는 구조였습니다.

하지만 모델 summary를 확인했을 때 Flatten 이후 큰 feature vector가 LSTM에 그대로 입력되면서 파라미터 수가 크게 증가하는 문제가 있었습니다. 또한 학습 과정에서는 이미지 로딩과 전처리 파이프라인도 병목이 되어 학습 시간이 길어졌습니다.

## 6. Raspberry Pi 실주행 병목 분석

학습 정확도만 보면 CNN-LSTM 구조가 의미 있을 수 있었지만, 실제 차량에 올렸을 때는 Raspberry Pi에서의 추론 지연이 더 큰 문제가 되었습니다. 실주행에서는 모델이 조금 더 정확한 것보다 조향 명령이 얼마나 빠르고 일정하게 나오는지가 더 중요했습니다.

여러 해상도 축소와 LSTM 구조 실험을 거쳤지만, 실제 주행에서는 예측 시간이 길고 반응성이 떨어지는 문제가 남았습니다. 이 과정에서 최종 병목은 모델 복잡도와 추론 지연이라고 판단했습니다.

## 7. 최종 해결: NVIDIA-style CNN + TFLite INT8 양자화

최종적으로는 CNN-LSTM 구조를 버리고 NVIDIA End-to-End Driving 스타일의 단일 프레임 CNN 구조로 단순화했습니다. 이 구조는 시계열 모델보다 단순하지만 Raspberry Pi에서 빠르게 추론할 수 있고, 실제 조향 반응성이 좋았습니다.

이후 CNN 모델을 TensorFlow Lite로 변환하고 INT8 양자화를 적용했습니다. 양자화를 통해 모델 크기와 추론 시간을 줄였고, 결과적으로 차량의 실시간 반응성이 개선되었습니다.

## 8. 센서 및 실주행 통합

최종 주행 코드에는 카메라 입력 기반 조향뿐만 아니라 초음파 센서를 이용한 장애물 정지 기능도 포함했습니다. 일정 거리 이내에 장애물이 감지되면 모터를 정지시키고, 시퀀스 버퍼를 사용하는 모델의 경우 버퍼를 초기화해 잘못된 입력이 이어지지 않도록 했습니다.

또한 Raspberry Pi에서 USB 카메라가 OpenCV로 불안정하게 열리는 문제를 해결하기 위해 MJPG 포맷 강제 지정과 GStreamer fallback 방식도 테스트했습니다.

## 9. 결과

차량 무게중심 조정, 모터 제어 튜닝, 데이터 수집, 모델 구조 실험, Raspberry Pi 추론 최적화, TFLite INT8 양자화, 초음파 정지 기능 통합을 거쳐 최종적으로 트랙 10바퀴를 빠른 시간 안에 완주했습니다. 결과적으로 캡스톤 자율주행 평가에서 1등을 달성했습니다.

## 10. 면접 답변 요약

> 초기에는 차량의 무게중심과 후륜 특성 때문에 사람이 조종해도 코너를 돌기 어려웠습니다. 먼저 무게중심을 후방으로 이동시키고 모터 제어 값을 조정해 차량 플랫폼을 안정화했습니다. 이후 CNN-LSTM 기반 시계열 모델까지 실험했지만, Raspberry Pi 실주행에서는 추론 지연이 더 큰 병목이라는 것을 확인했습니다. 그래서 최종적으로 NVIDIA-style CNN 단독 구조로 단순화하고 TFLite INT8 양자화를 적용해 실시간성을 확보했습니다. 그 결과 트랙 10바퀴 주행 평가에서 1등을 달성했습니다.
