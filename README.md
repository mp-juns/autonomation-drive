### 차량 문제 발견 및 해결

<p align="center">
  <img src="docs/images/hardware_weight_balance_02.jpeg" width="700" alt="무게중심 조정">
</p>

초기 차량은 다음과 같은 문제가 있었습니다.

- 차체 길이가 길었음
- 후륜 구동 방식이었음
- 무게중심이 중앙에 위치해 있었음

그 결과 코너 구간에서 회전이 원활하지 않았고, 사람이 직접 조이스틱으로 조종하는 경우에도 안정적인 주행이 어려웠습니다.

당시 남은 시간이 많지 않아 차량을 새롭게 설계하고 출력하기 어려웠기 때문에, 차량 구조를 유지한 상태에서 문제를 해결해야 했습니다.

해결 방법은 다음과 같습니다.

- 배터리 위치 변경
- 무게중심 후방 이동
- 모터 제어값 조정
- 회전 시 좌우 바퀴 속도 차 확대

이를 통해 차량의 회전 성능을 개선하였고 안정적인 주행이 가능해졌습니다.

## 데이터 수집

MIT App Inventor를 이용하여 스마트폰 조이스틱 애플리케이션을 제작하였습니다.

해당 애플리케이션을 이용하여 차량을 직접 주행시키면서 다음 데이터를 수집하였습니다.

- 카메라 이미지
- 조향 데이터
- 주행 데이터

차량이 안정화된 이후 수집한 데이터는 모델 학습 품질 향상에 큰 도움이 되었습니다.

## 딥러닝 모델 개발

### CNN-LSTM 모델 실험

초기에는 연속 프레임 정보를 활용하기 위해 CNN-LSTM 구조를 사용하였습니다.

```text
연속 이미지
↓
CNN
↓
LSTM
↓
조향값 예측
```

이를 통해 차량의 주행 흐름을 반영하고자 하였습니다.

---

### 문제 분석

하지만 Raspberry Pi 환경에서는 추론 지연이 발생했습니다.

실험 결과 약 32ms 수준의 추론 지연이 발생하였고, 실제 주행에서는 이 지연이 차량 반응성을 저하시켰습니다.

이를 통해 다음 사실을 확인할 수 있었습니다.

> 정확도보다 실시간성이 더 중요한 병목이었다.

---

### 최종 모델

최종적으로는 NVIDIA End-to-End Driving CNN 구조를 참고하여 단일 프레임 CNN 모델로 변경하였습니다.

```text
카메라 이미지
↓
CNN
↓
조향값 예측
```

모델을 단순화하여 추론 속도를 향상시키고 실시간 제어 성능을 확보하였습니다.

---

## TensorFlow Lite 최적화

프로젝트 후반부에서 가장 중요한 작업은 Raspberry Pi 환경 최적화였습니다.

CNN 모델을 TensorFlow Lite로 변환한 후 INT8 양자화를 적용하였습니다.

적용 기술은 다음과 같습니다.

- TensorFlow Lite
- INT8 Quantization

이를 통해 다음 효과를 얻었습니다.

- 모델 크기 감소
- 메모리 사용량 감소
- 추론 속도 향상
- 실시간 주행 성능 향상

결과적으로 모델 복잡도를 높이는 것보다 양자화를 통한 실시간성 확보가 훨씬 큰 효과를 보였습니다.
